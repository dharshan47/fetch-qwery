# fetch-qwery

Welcome to the **fetch-qwery** documentation! 🚀

`fetch-qwery` is a modern, high-performance data fetching library that provides a **zero-config**, developer-friendly API. It handles caching, state management, and parallel fetching internally, allowing you to use it directly with no complex setup, boilerplate, or providers.

## Key Features

- 🦄 **True Zero Config**: No providers or `QueryClient` setup—just import and fetch.
- ⚡ **Lightning Fast**: Highly optimized for performance with minimal bundle footprint.
- 🔄 **Intelligent Caching**: Seamless data persistence and automatic background updates.
- 🧬 **Parallel Fetching**: Native support for fetching multiple resources as a single unit.
- 🛡️ **Race Condition Protection**: Handles async timing issues automatically.
- 🌑 **Next.js & SSR Ready**: Built for modern React and Next.js (Server & Client components).

## Why fetch-qwery?

Standard `fetch` is powerful but managing loading states, error handling, and caching manually is repetitive and error-prone. `fetch-qwery` simplifies this process, giving you a powerful, typesafe hook that works out of the box.
